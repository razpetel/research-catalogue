# Railway Research Report

## Overview

Railway is a full-stack cloud platform for deploying web apps, databases, and services with automatic scaling and zero-config setup. Founded in 2020 and emerging from stealth with a $20M raise from Redpoint Ventures in 2022, Railway has positioned itself as the modern successor to Heroku—offering a developer-first experience that abstracts away infrastructure complexity.

The company just announced a **$100M Series B** (January 22, 2026), bringing total funding to $130M+. Led by TQ Ventures with participation from FPV Ventures, Redpoint, and Unusual Ventures, this funding validates Railway's rapid growth: **2+ million developers**, **200,000 new users monthly**, and **31% Fortune 500 adoption**. Customers report 10x developer velocity improvements and up to 65% cost savings versus traditional cloud platforms.

Railway's core value proposition is "Zero-Ops"—push code, get a running app. The platform handles builds (via their open-source Nixpacks), deployments, databases, networking, and scaling. Their recent focus on AI/agent integration (MCP server, agent skills) positions them for the emerging wave of AI-native applications.

## Deploy Button & Templates

Railway's Deploy Button enables one-click deployment of full-stack applications from GitHub repositories. It powers the template marketplace of 2,000+ templates and includes a creator kickback program that has paid **over $1 million** to template authors.

### Deploy Button Formats

**Template ID (Recommended):**
```markdown
[![Deploy on Railway](https://railway.com/button.svg)](https://railway.com/new/template/{TEMPLATE_CODE}?referralCode={YOUR_CODE})
```

**GitHub Repository (Dynamic):**
```markdown
[![Deploy on Railway](https://railway.com/button.svg)](https://railway.com/new/template?template={REPO_URL}&envs={VARS})
```

### URL Parameters

| Parameter | Required | Description |
|-----------|----------|-------------|
| `template` | Yes* | URL-encoded GitHub repo URL |
| `envs` | No | Comma-separated required env vars |
| `optionalEnvs` | No | Comma-separated optional env vars |
| `{VarName}Desc` | No | Description for each variable |
| `referralCode` | No | Creator referral code for kickbacks |

*For repo-based deploys. Template ID deploys use the path instead.

### Configuration Files

**railway.json (Single Service):**
```json
{
  "$schema": "https://railway.app/railway.schema.json",
  "build": { "builder": "NIXPACKS" },
  "deploy": {
    "startCommand": "npm start",
    "healthcheckPath": "/health",
    "restartPolicyType": "ON_FAILURE"
  }
}
```

**railway.toml (Multi-Service):**
```toml
[build]
builder = "NIXPACKS"

[deploy]
numReplicas = 1
restartPolicyType = "ON_FAILURE"
restartPolicyMaxRetries = 5
```

**Builder Options:** `NIXPACKS` (auto-detection), `RAILPACK` (optimized), `DOCKERFILE` (custom)

## Recommendation

**For hobby/indie projects:** Excellent choice. $5/month gets you a build pipeline, PR environments, custom domains, and databases—hard to beat for rapid prototyping.

**For production workloads:** Proceed with caution. Performance concerns on Reddit are real, especially for latency-sensitive applications. The credit-based model requires monitoring. Consider Render for more predictable production environments.

**For AI applications:** Strong positioning. Railway's MCP server and template ecosystem (Ollama, vector databases, etc.) make it attractive for AI-native apps requiring multiple services.

**For template creators:** Attractive economics. 25-50% revenue share with $1M+ already paid out. Use template IDs over repo URLs, include railway.json for predictable builds, and add referralCode to earn kickbacks.

**Wait and see:** The $100M funding will accelerate growth. Watch for enterprise SLAs, improved performance, and expanded regions before committing critical production workloads.
