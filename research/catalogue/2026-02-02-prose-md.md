# OpenProse (prose.md) Research Report

## Overview

OpenProse is a novel programming language for AI agent orchestration that takes a radically different approach: instead of external frameworks coordinating agents, the AI session itself becomes the runtime. The project is implemented entirely as Markdown files—the core specification (`prose.md`) describes a virtual machine that AI systems "simulate" when they read it.

Created by maintainer irl-dan and released in early January 2026, OpenProse has quickly gained traction with 728 GitHub stars and 63 forks in its first month. The project supports multiple AI coding assistants (Claude Code, OpenCode, Amp) with zero external dependencies. Its unique innovation is treating the AI session as an "Inversion of Control container"—you declare agents and control flow, the session wires them up.

The key differentiator is the "Fourth Wall" syntax (`**...**`) which allows natural language conditions that the AI evaluates semantically, e.g., `loop until **the code is production ready**`. This bridges the gap between rigid programming constructs and flexible AI judgment.

### Language Features

**Implemented:**
- Agent definitions with model/skills/permissions
- Sequential and parallel execution blocks
- Variable binding (let/const) with context passing
- Iteration: repeat, for-each, parallel for-each, unbounded loops
- AI-evaluated conditions: `loop until **condition**`, `loop while **condition**`
- Pipelines: map, filter, reduce, parallel map
- Error handling: try/catch/finally, throw, retry with backoff
- Program composition: use statements, input/output declarations

**Example Program:**
```prose
agent researcher:
  model: sonnet
  skills: ["web-search"]

agent writer:
  model: opus

parallel:
  research = session: researcher
    prompt: "Research quantum computing breakthroughs"
  competitive = session: researcher
    prompt: "Analyze competitor landscape"

loop until **the draft meets publication standards** (max: 3):
  session: writer
    prompt: "Write and refine the article"
    context: { research, competitive }
```
