# BMAD + ralph-tui Integration Guide

## Goal

Use BMAD-METHOD for structured planning (Phases 1-3), then hand off to ralph-tui for autonomous execution (Phase 4) -- replacing BMAD's built-in dev loop with ralph's autonomous agent loop and fresh-context-per-iteration model.

## Prerequisites

```bash
npm install -g bmad-method    # v6+
bun install -g ralph-tui      # v0.11+
```

## Directory Layout

```
your-project/
├── _bmad/                          # BMAD agents, workflows, config
├── _bmad-output/
│   ├── planning-artifacts/         # BMAD Phase 1-3 output
│   │   ├── prd.md
│   │   ├── architecture.md
│   │   └── epics.md                # or sharded epic-1.md, epic-2.md
│   └── implementation-artifacts/   # BMAD sprint tracking (optional with ralph)
├── .ralph-tui/
│   ├── config.toml                 # ralph-tui config
│   └── session.json                # ralph-tui state (auto-generated)
├── prd.json                        # ralph-tui task file (generated by bridge script)
├── .claude/
│   ├── settings.json               # hooks config
│   └── hooks/
│       └── bmad-to-ralph.sh        # bridge script
└── CLAUDE.md
```

## Workflow

### Phase 1-3: BMAD Planning (human-in-the-loop)

Run in fresh Claude Code sessions. Each phase produces markdown artifacts.

```
/bmad-bmm-create-prd              # Phase 2: PRD (required)
/bmad-bmm-create-architecture     # Phase 3: Architecture (required)
/bmad-bmm-create-epics-and-stories # Phase 3: Epics + stories (required)
/bmad-bmm-check-implementation-readiness  # Phase 3: Cross-doc validation
```

Output: `_bmad-output/planning-artifacts/epics.md` with stories in this format:

```markdown
## Epic 1: User Authentication

### Story 1.1: Login Flow
As a user, I want to log in so that I can access my account.

**Acceptance Criteria:**
**Given** valid credentials
**When** I submit the login form
**Then** I am redirected to the dashboard
```

### Bridge: Convert BMAD stories to ralph-tui prd.json

Save this as `.claude/hooks/bmad-to-ralph.sh`:

```bash
#!/bin/bash
# Converts BMAD epics.md → ralph-tui prd.json
# Usage: .claude/hooks/bmad-to-ralph.sh [epic-number]

set -euo pipefail

EPICS_FILE="_bmad-output/planning-artifacts/epics.md"
OUTPUT="prd.json"
EPIC_FILTER="${1:-}"  # optional: only convert specific epic

if [ ! -f "$EPICS_FILE" ]; then
  echo "Error: $EPICS_FILE not found. Run BMAD planning first." >&2
  exit 1
fi

# Extract project name from first heading
PROJECT_NAME=$(grep -m1 '^# ' "$EPICS_FILE" | sed 's/^# //' | sed 's/ - Epic Breakdown//')

# Parse stories into prd.json using jq
# Each "### Story N.M: Title" becomes a userStory
python3 -c "
import re, json, sys

with open('$EPICS_FILE') as f:
    content = f.read()

stories = []
epic_filter = '$EPIC_FILTER'

# Find all epics
epic_pattern = r'## Epic (\d+): (.+?)(?=\n## Epic \d+:|\Z)'
story_pattern = r'### Story (\d+\.\d+): (.+?)(?=\n### Story \d+\.\d+:|\n## Epic \d+:|\Z)'
ac_pattern = r'\*\*(?:Given|When|Then|And)\*\*\s*(.+?)(?=\n\*\*(?:Given|When|Then|And)\*\*|\n###|\n##|\Z)'

for epic_match in re.finditer(epic_pattern, content, re.DOTALL):
    epic_num = epic_match.group(1)
    if epic_filter and epic_num != epic_filter:
        continue

    epic_text = epic_match.group(0)

    for story_match in re.finditer(story_pattern, epic_text, re.DOTALL):
        story_id = story_match.group(1)
        story_title = story_match.group(2).strip()
        story_text = story_match.group(0)

        # Extract description (As a... line)
        desc_match = re.search(r'(As a .+?(?:so that .+?\.))', story_text, re.DOTALL)
        description = desc_match.group(1).strip() if desc_match else ''

        # Extract acceptance criteria
        criteria = re.findall(ac_pattern, story_text, re.DOTALL)
        criteria = [c.strip() for c in criteria]

        # Derive dependency: story X.Y depends on X.(Y-1) if Y > 1
        parts = story_id.split('.')
        depends = []
        if int(parts[1]) > 1:
            depends.append(f'US-{parts[0]}{int(parts[1])-1:02d}')

        stories.append({
            'id': f'US-{parts[0]}{int(parts[1]):02d}',
            'title': story_title,
            'description': description,
            'acceptanceCriteria': criteria if criteria else [story_title],
            'priority': len(stories) + 1,
            'passes': False,
            'notes': '',
            'dependsOn': depends
        })

prd = {
    'name': '$PROJECT_NAME' or 'BMAD Project',
    'branchName': 'ralph/' + re.sub(r'[^a-z0-9]+', '-', ('$PROJECT_NAME' or 'bmad-project').lower()).strip('-'),
    'description': 'Generated from BMAD planning artifacts',
    'userStories': stories
}

json.dump(prd, sys.stdout, indent=2)
" > "$OUTPUT"

STORY_COUNT=$(jq '.userStories | length' "$OUTPUT")
echo "Converted $STORY_COUNT stories to $OUTPUT"
```

Make executable: `chmod +x .claude/hooks/bmad-to-ralph.sh`

### Phase 4: ralph-tui Execution (autonomous)

```bash
# Convert BMAD artifacts to prd.json
.claude/hooks/bmad-to-ralph.sh

# Optional: convert only one epic at a time
.claude/hooks/bmad-to-ralph.sh 1

# Run ralph-tui
ralph-tui run --prd ./prd.json
```

## ralph-tui Configuration

`.ralph-tui/config.toml`:

```toml
agent = "claude"
tracker = "json"
maxIterations = 0              # unlimited -- let it finish
autoCommit = false             # commit manually or via CI

[agentOptions]
model = "claude-opus-4-6"      # or sonnet for speed

[errorHandling]
strategy = "skip"              # skip broken stories, come back later
maxRetries = 3

[rateLimitHandling]
enabled = true
maxRetries = 3
baseBackoffMs = 5000

[notifications]
enabled = true
```

## Hooks for Automation

Add to `.claude/settings.json` for workflow automation:

```json
{
  "hooks": {
    "Stop": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "bash -c 'INPUT=$(cat); if [ \"$(echo \"$INPUT\" | jq -r .stop_hook_active)\" = \"true\" ]; then exit 0; fi; if [ -f \"_bmad-output/planning-artifacts/epics.md\" ] && [ ! -f \"prd.json\" ]; then echo \"BMAD planning complete. Run: .claude/hooks/bmad-to-ralph.sh\" >&2; fi; exit 0'",
            "timeout": 5
          }
        ]
      }
    ],
    "Notification": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "osascript -e 'display notification \"Claude Code needs attention\" with title \"BMAD + Ralph\"'"
          }
        ]
      }
    ]
  }
}
```

## CLAUDE.md Additions

Append to your project's `CLAUDE.md`:

```markdown
## Development Workflow

This project uses BMAD-METHOD for planning and ralph-tui for execution.

### Planning (Phases 1-3)
- Run BMAD slash commands in fresh sessions
- All planning artifacts go to `_bmad-output/planning-artifacts/`
- Always run `/bmad-bmm-check-implementation-readiness` before implementation

### Implementation (Phase 4)
- Convert stories: `.claude/hooks/bmad-to-ralph.sh`
- Execute: `ralph-tui run --prd ./prd.json`
- Task format: prd.json with userStories[] (not BMAD story files)
- Each story must have acceptance criteria that map to verifiable checks

### Conventions
- Branch naming: `ralph/<feature-kebab-case>`
- One epic at a time for large projects
- Run quality gates before marking stories complete
```

## Iterative Development (Epic-by-Epic)

For large projects, plan and implement one epic at a time:

1. Complete BMAD Phases 1-3 (full PRD, architecture, all epics)
2. Convert Epic 1 only: `.claude/hooks/bmad-to-ralph.sh 1`
3. Run ralph-tui until Epic 1 stories are all `passes: true`
4. Review, merge Epic 1 branch
5. Convert Epic 2: `.claude/hooks/bmad-to-ralph.sh 2`
6. Repeat

This keeps ralph-tui's context focused and avoids overwhelming it with the full project scope.

## Re-planning After Partial Completion

If you need to revise the plan after some stories are done:

1. Note which `US-*` IDs have `"passes": true` in `prd.json`
2. Run BMAD planning updates (`/bmad-bmm-edit-prd`, re-run epics)
3. Re-run the bridge: `.claude/hooks/bmad-to-ralph.sh`
4. Manually set `"passes": true` for already-completed stories in the new `prd.json`
5. Resume: `ralph-tui resume` or `ralph-tui run --prd ./prd.json`

## Pitfalls

| Pitfall | Mitigation |
|---------|------------|
| BMAD personas refused by Claude ("my core identity is fixed") | Use fresh sessions; avoid stacking multiple agent personas |
| Context exhaustion during BMAD planning | One phase per session; use `/bmad-help` to check progress |
| ralph-tui runaway (280 calculators incident) | Set `maxIterations` to a sane limit for first runs |
| Acceptance criteria too vague for ralph | Add concrete quality gates: `pnpm typecheck passes`, `pnpm test passes` |
| Story dependencies not mapped | Bridge script auto-derives sequential deps; add cross-epic deps manually |
| prd.json schema rejected | No wrapper objects, must use `userStories` (not `tasks`), `passes` (not `status`) |
